# CHAP1 to R

Author: Christopher Odoom

R implementation of the CHAP1 / DeepPostures CNN-BiLSTM posture prediction
pipeline. The active R files still use `chap2r` in their names for historical
reasons, but this project targets the CHAP1/MSSE-2021 model family.

## Blueprint

This project ports the CHAP1 posture prediction path into R while keeping
compatibility with the original DeepPostures preprocessing outputs and trained
model checkpoints.

Current scope:

- Read preprocessed day-level H5 files generated by the Python DeepPostures
  preprocessing pipeline.
- Load converted CHAP1 checkpoints from R-native `.rds` files.
- Run the CNN-BiLSTM model in R `torch`.
- Write per-day posture prediction CSV files, with optional labels, segment IDs,
  and probabilities.

Planned scope:

- Add an R implementation of the preprocessing step. Once that exists, the
  project can choose the most convenient saved format for model prediction
  inputs instead of depending only on Python-generated H5 files.

## Project Layout

```text
R/
  chap2r_predict_from_h5.R       command-line prediction script
  chap2r_utils.R                 H5 reading, segmentation, model loading, CSV output
  chap2r_model.R                 R torch CNN-BiLSTM architecture
  rhelper/
    check_setup.R                optional environment and file diagnostic
    check_h5_reader.R            optional H5 reader diagnostic
    test_run_prediction.R        optional one-day smoke test
    convert_pt_to_rtorch.R       converts Python .pth checkpoints to R .rds

CHAP_trained_models/
  pre-trained-models-rtorch/     converted R checkpoints used by prediction

data/
  preprocessed/                  input H5 files; one small sample day is included
  predictions/                   default prediction output directory
```

## Inputs

The prediction script expects preprocessed H5 files under a subject directory:

```text
data/preprocessed/
  62193/
    2000-01-07/
      2000-01-07.h5
```

The reader searches recursively below `data/preprocessed/<subject-id>`, so day
folders are supported.

Only one small sample H5 day is included in this repository for testing:

```text
data/preprocessed/62193/2000-01-07/2000-01-07.h5
```

Full-study preprocessed H5 data can become large quickly and is ignored by
`.gitignore` by default.

For now, generate these H5 files with the Python CHAP / DeepPostures
preprocessing implementation. The DeepPostures documentation describes
`pre_process_data.py`, including the required `--gt3x-dir` and
`--pre-processed-dir` arguments:

```text
https://adalabucsd.github.io/DeepPostures/getting_started/preprocess/
```

An R version of this preprocessing step is planned, but it is not implemented in this repository yet.

Each H5 file should contain:

```text
time       numeric vector, one Unix timestamp per 10-second window
data       numeric array, expected as [window, 100, 3]
sleeping   integer vector, 1 means sleep window
non_wear   integer vector, 1 means non-wear window
label      integer vector, -1 no label, 0 sitting, 1 not-sitting
```

## Models

Supported model names:

```text
CHAP                      (ensemble of CHAP_A, CHAP_B, CHAP_C)
CHAP_A
CHAP_B
CHAP_C
CHAP_ALL_ADULTS
CHAP_CHILDREN
CHAP_AUSDIAB
```

The default model is `CHAP_ALL_ADULTS`.

If you set `--model-name CHAP`, predictions are computed as an ensemble of
`CHAP_A`, `CHAP_B`, and `CHAP_C` by averaging their sigmoid probabilities and
then applying the `--threshold`.

Default checkpoints are loaded from:

```text
CHAP_trained_models/pre-trained-models-rtorch/<model-name>.rds
```

## Environment

### R Prediction Environment

The normal prediction pipeline only needs R packages. It does not need Python if
the converted `.rds` checkpoints already exist.

Required R packages for normal prediction:

```r
torch
abind
rhdf5
```

The supported H5 reader is `rhdf5`. It is a Bioconductor package and installs
`Rhdf5lib` and `rhdf5filters`, which provide the HDF5 backend used to read
preprocessed `.h5` files. Installation can print many compiler checks on macOS;
that output is normal for native HDF5 packages.

Install the R prediction environment:

```r
install.packages(c("torch", "abind", "BiocManager"))
BiocManager::install("rhdf5", ask = FALSE, update = FALSE)
torch::install_torch()
```

# Note:
# - `install.packages("torch")` installs the R package (the R interface to Torch).
# - `torch::install_torch()` downloads and installs the actual Torch (LibTorch) backend binaries needed for computation.
#   Both steps are required for full functionality.

`hdf5r` is still supported by the code as an optional fallback if it is already
available, but it is not the recommended setup path for this project because it
can be sensitive to local Homebrew or conda HDF5 versions.

After installing packages, run the setup diagnostics:

```bash
Rscript R/rhelper/check_setup.R
Rscript R/rhelper/check_h5_reader.R
```

If a failed H5-reader install leaves `renv` in a bad state, restart R and remove
only the partial package named in the warning. For example:

```r
remove.packages("Rhdf5lib")
```

Then rerun the `rhdf5` install command above.

If you want a reproducible project-local R environment, use `renv`:

```r
install.packages("renv")
renv::init()
```

After `renv::init()`, restart the R session if prompted. Then install packages
into the activated project library:

```r
install.packages(c("torch", "abind", "BiocManager"))
BiocManager::install("rhdf5", ask = FALSE, update = FALSE)
torch::install_torch()
renv::snapshot()
```

Future users can restore that environment with:

```r
renv::restore()
```

If prediction fails before model inference, check H5 reading directly:

```bash
Rscript R/rhelper/check_h5_reader.R
```

You can also pass a specific H5 file:

```bash
Rscript R/rhelper/check_h5_reader.R data/preprocessed/62193/2000-01-07/2000-01-07.h5
```

### Optional Python Conversion Environment

Python is only needed if someone wants to regenerate the `.rds` checkpoints from
the original Python `.pth` files. Those `.pth` files are optional conversion
inputs and are not included in this GitHub-ready tree by default.

The conversion script uses R `reticulate` to load Python checkpoints with
Python `torch`, then saves R-native `.rds` files.

Required Python libraries:

```text
torch
numpy
```

Required R package for conversion:

```r
reticulate
```

Install it with:

```r
install.packages("reticulate")
```

Example Python setup with conda:

```bash
conda create -n chap-convert python=3.10 numpy pytorch -c pytorch -c conda-forge
CHAP_CONDA_ENV=chap-convert Rscript R/rhelper/convert_pt_to_rtorch.R
```

Example Python setup with `venv`:

```bash
python -m venv .venv-chap-convert
source .venv-chap-convert/bin/activate
python -m pip install --upgrade pip
python -m pip install torch numpy
CHAP_PYTHON="$PWD/.venv-chap-convert/bin/python" Rscript R/rhelper/convert_pt_to_rtorch.R
```

If no environment variable is set, `reticulate` uses its normal Python
discovery. To pin the Python environment, set one of these:

```bash
CHAP_PYTHON=/path/to/python Rscript R/rhelper/convert_pt_to_rtorch.R
CHAP_CONDA_ENV=chap-convert Rscript R/rhelper/convert_pt_to_rtorch.R
```

To regenerate the checkpoints:

```bash
mkdir -p CHAP_trained_models/pre-trained-models-pt
# Copy or download the original .pth files into that directory first.
Rscript R/rhelper/convert_pt_to_rtorch.R
```

## Test Run

Use the bundled sample day to check that the R prediction pipeline is working before running a full subject.

First check the environment and H5 reader:

```bash
Rscript R/rhelper/check_setup.R
Rscript R/rhelper/check_h5_reader.R
```

Then run the one-day prediction smoke test:

```bash
Rscript R/rhelper/test_run_prediction.R
```

The test script:

- Copies `data/preprocessed/62193/2000-01-07` into a temporary one-day input
  directory.
- Runs `CHAP_ALL_ADULTS` with `padding = "drop"` and `threshold = 0.5`.
- Writes segment IDs, H5 labels, probabilities, and predictions.
- Stops with an error if the expected CSV is not created.

Expected CSV:

```text
data/tmp/test_predictions/62193/CHAP_ALL_ADULTS/2000-01-07.csv
```

## Run Predictions

Example command for subject `62193`:

```bash
Rscript R/chap2r_predict_from_h5.R \
  --preprocessed-dir data/preprocessed \
  --subject-id 62193 \
  --model-name CHAP_ALL_ADULTS \
  --padding drop \
  --threshold 0.5 \
  --down-sample-frequency 10 \
  --output-file data/predictions/62193/CHAP_ALL_ADULTS \
  --segment \
  --output-label \
  --output-prob \
  --overwrite
```

This writes one CSV per calendar day:

```text
data/predictions/62193/CHAP_ALL_ADULTS/2000-01-07.csv
data/predictions/62193/CHAP_ALL_ADULTS/2000-01-08.csv
```

If `--output-file` is omitted, the default output directory is:

```text
data/predictions/<subject-id>/<model-name>
```

### Batch Processing Hint

Most studies have many participant folders under `data/preprocessed`. For a full
run, create a small project-specific batch script that loops over those subject
IDs and calls `R/chap2r_predict_from_h5.R` once per participant. Keep each
participant's output in its own directory and keep test/temp outputs under
`data/tmp` so cleanup is easy.

Example shell wrapper:

```bash
mkdir -p data/predictions

for subject_dir in data/preprocessed/*; do
  subject_id="$(basename "$subject_dir")"

  Rscript R/chap2r_predict_from_h5.R \
    --preprocessed-dir data/preprocessed \
    --subject-id "$subject_id" \
    --model-name CHAP_ALL_ADULTS \
    --padding drop \
    --threshold 0.5 \
    --down-sample-frequency 10 \
    --output-file "data/predictions/${subject_id}/CHAP_ALL_ADULTS" \
    --segment \
    --output-label \
    --output-prob \
    --overwrite
done
```

For repeatable analyses, save a wrapper like this under your own `scripts/`
folder and adjust the model name, threshold, and output options there.

## Command Arguments

`--preprocessed-dir`
: Root directory containing subject folders. Use `data/preprocessed` for this
project.

`--subject-id`
: Subject folder name below `--preprocessed-dir`.

`--model-name`
: CHAP1 model name. Defaults to `CHAP_ALL_ADULTS`.

`--checkpoint`
: Optional checkpoint override. If omitted, the script uses
`CHAP_trained_models/pre-trained-models-rtorch/<model-name>.rds`.

`--output-file`
: Output directory for per-day CSV files.

`--down-sample-frequency`
: Frequency of the preprocessed data. Default is `10`.

`--padding`
: How incomplete BiLSTM sequences are handled. Supported values are `drop`,
`zero`, and `wrap`.

`--threshold`
: Sigmoid probability threshold used to write the `prediction` label. Defaults
to `0.5`.

`--segment`
: Include a zero-based `segment` column.

`--output-label`
: Include labels from the H5 file.

`--output-prob`
: Include sigmoid probabilities. If omitted, the CSV still includes
`prediction`, but not `probability`.

`--overwrite`
: Replace existing day CSVs on first write for each day. This is the default.

`--append`
: Append to existing day CSVs.

## Output

With `--segment --output-label`, output CSV columns are:

```text
segment,timestamp,label,prediction
```

With `--segment --output-label --output-prob`, output CSV columns are:

```text
segment,timestamp,label,probability,prediction
```

Labels and predictions are written as:

```text
no-label
sitting
not-sitting
```

Predictions are always written. By default, probabilities below `0.5` are
written as `sitting`, and probabilities greater than or equal to `0.5` are
written as `not-sitting`. Use `--threshold` to change this decision rule.

## Segmentation

The pipeline reads all H5 files for a subject in sorted order, creates
continuous awake/wear segments, and breaks a segment whenever `sleeping == 1` or
`non_wear == 1`. Output rows are split back into calendar-day CSV files by
timestamp.

- Run commands from the project root so relative paths resolve correctly.
- The R implementation is binary-only and expects CHAP labels `0` for sitting
  and `1` for not-sitting.
- The checkpoint loader automatically handles Python/R `torch` naming
  differences for bidirectional LSTM weights.
